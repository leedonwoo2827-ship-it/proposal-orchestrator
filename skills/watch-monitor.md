# 워치/모니터링 기능

## Purpose
현재 프로젝트 상태를 점검하고 설명하는 역할을 수행한다.
진행 상황 보고, 누락 확인, 보완 제안까지 포괄한다.

## When to Use
- 사용자가 "진행 상황 보여줘", "어디까지 했어?" 라고 물을 때
- 제안서 작성 중간에 전체 상태를 점검할 때
- 최종 검수/리뷰 단계에서

## 상태 조회 방법

project_id 기반으로 다음 정보를 조회한다:
- 현재까지 작성된 섹션 목록
- 각 섹션의 작성 상태 (완료 / 부분 작성 / 미작성)
- 생성된 이미지 목록 (용도, 경로, 생성 일시)
- HWPX 출력 여부

## 상태 보고 형식

### 텍스트 상태
```
[제안서 작성 현황]
- 완료: 1. 개요, 2. 시장 분석, 3. 서비스 소개
- 작성 중: 4. 기술 아키텍처 (약 60% 완료)
- 미작성: 5. 수익 모델, 6. 추진 일정, 7. 기대 효과
- 전체 진행률: 약 45%
- 현재 글자 수: 약 45,000자 / 목표 100,000자
```

### 이미지 상태
```
[이미지 생성 현황]
- 표지용: 1개 완료
- 슬라이드용: 2/4개 완료 (시장 분석, 서비스 소개)
- 미생성: 기술 아키텍처, 기대 효과 섹션용 이미지
```

### 종합 보고
```
[프로젝트 종합 현황]
- Project ID: PROP-2026-001
- 목표: 10만자 제안서 + 이미지 5장
- 텍스트: 45% 완료
- 이미지: 60% 완료 (3/5장)
- 예상 남은 작업: 본문 55,000자 작성 + 이미지 2장 생성 + HWPX 출력
```

## 워치 기반 제안

부족한 부분에 대해 구체적으로 제안한다:

### 텍스트 보완 제안
- "수익 모델 섹션이 아직 미작성입니다. 작성을 진행할까요?"
- "시장 분석 섹션의 분량이 목표(15,000자) 대비 60%입니다. 보완할까요?"
- "기술 아키텍처 섹션에 비교표를 추가하면 설득력이 높아집니다."

### 이미지 보완 제안
- "시장 분석 섹션에 그래프형 인포그래픽 이미지를 추가하는 것을 제안합니다."
- "기대 효과 섹션에 비전 이미지가 없습니다. 1장 생성할까요?"

### 전체 구조 제안
- "현재 목차에 '리스크 관리' 섹션이 빠져 있습니다. 추가할까요?"
- "경쟁사 분석이 약합니다. 참조 데이터를 추가 분석할까요?"
