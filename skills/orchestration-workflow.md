# 제안서 오케스트레이션 워크플로

## Purpose
사용자의 자연어 요청을 해석하여 제안서 작성, 이미지 생성, 진행 상황 보고를 단계적으로 수행한다.
등록된 MCP 툴(hwpx-writer, text2img-mcp, filesystem 등)을 적절한 순서와 방식으로 호출하여
제안서 텍스트 작성 + 이미지 생성 + 결과 요약/워치 기능을 제공한다.

## When to Use
- 사용자가 제안서 작성을 요청할 때
- 제안서와 이미지를 동시에 생성해야 할 때
- 프로젝트 진행 상황을 확인하거나 검토할 때
- 기존 제안서를 수정/보완할 때

## 입력 파싱 규칙

사용자 메시지에서 다음 요소를 추출한다:

### 작업 유형
- **제안서 전체 생성**: "제안서 만들어줘", "양식으로 제안서 작성해줘"
- **일부 섹션 보완**: "시장 분석 부분 보완해줘", "결론 다시 써줘"
- **이미지 생성**: "표지 이미지 만들어줘", "슬라이드 이미지 3장"
- **워치/점검**: "진행 상황 보여줘", "빠진 섹션 확인해줘"

### 프로젝트 식별
- project_id (명시적으로 제공된 경우)
- 파일 경로
- "이전에 만들었던 ~ 제안서" 같은 설명

### 텍스트 옵션
- 목표 글자 수: 5만 / 10만 / 15만 / 20만자
- 톤/스타일: 금융, 공공, 교육, IT, 건설 등

### 이미지 옵션
- 생성 개수: 1장, 3개, 여러 장
- 컨셉/스타일: 미니멀, 인포그래픽, 실사, 일러스트
- 용도: 표지(cover), 슬라이드(slide), 요약(summary), 결론(conclusion)

### 출력 형식
- HWPX 파일 생성 여부
- 요약/목차만 볼지 여부

## 오케스트레이션 규칙

### 공통 규칙
1. 가능한 한 동일 project_id를 사용하며, 모든 툴 호출에 전달한다.
2. 툴 호출 전: 입력 파라미터(글자 수, 이미지 개수 등)를 사용자와 재확인한다.
3. 툴 호출 후: 결과 요약(텍스트/이미지/파일 경로) 및 다음 단계를 제안한다.

### 제안서 전체 생성 워크플로
1. **요청 파싱** → 작업 유형, 글자 수, 이미지 개수, 스타일 추출. project_id 없으면 새로 생성.
2. **양식 분석** → 양식 분석 툴 호출 → 섹션 구조/목차 JSON 획득.
3. **참조 데이터 분석** → 참조제안서/로데이터 분석 → 섹션별 핵심 요약.
4. **본문 작성** → 글자 수 옵션 포함하여 섹션별 텍스트 생성. 필요 시 서론→본론→결론 순 단계적 작성.
   - **출처 마커 필수**: rawdata에서 가져온 텍스트는 `{{red:텍스트}}`로, 참조문서에서 가져온 텍스트는 `{{green:텍스트}}`로 마킹한다.
   - AI가 새로 생성한 텍스트는 마커 없이 기본 상태로 둔다.
   - 마크다운 볼드(`**텍스트**`)는 사용하지 않는다 — hwpx-writer가 지원하지 않아 별표가 그대로 제거된다. 강조는 색상 마커 또는 헤딩 레벨(lv)로 대체한다.
5. **이미지 생성** → 이미지가 필요한 섹션/용도 파악. text2img-mcp로 prompt, project_id, 용도, 개수 전달.
6. **결과 묶기/내보내기** → hwpx-writer로 본문 텍스트(proposal-body.md) + 이미지 정보 + 스타일 옵션 전달하여 최종 파일 생성.
   - `{{red:}}` / `{{green:}}` 마커는 hwpx-writer가 자동으로 색상으로 변환한다. 별도 변환 작업 불필요.
   - proposal-style-guide의 "출처별 색상 표시 규칙"을 준수한다.

### 이미지 중심 워크플로
1. 텍스트 요약 또는 기존 project_id의 내용 활용
2. 이미지 프롬프트 자동 생성 및 다중 이미지 생성
3. 각 이미지에 대한 설명/캡션 제안

### 워치/점검 워크플로
1. project_id 기반 텍스트/이미지 상태 조회
2. 완료/부분/미작성 섹션 정리하여 보고
3. 이미지 커버리지 분석
4. 수정 제안 및 추가 작업 플랜 제시

## 대화 스타일
- 각 단계를 짧게 보고한다:
  - "1단계: 양식 분석 중입니다."
  - "2단계: 참조제안서에서 핵심 포인트 추출 완료."
  - "3단계: 본문 작성 후, 이미지 4개를 생성하겠습니다."
- MCP 툴 호출 전 중요한 설정을 사용자에게 확인 받는다.
- 결과 반환 시 무엇을 했는지 + 다음 선택지를 함께 제안한다.

## 주의사항
- 사용자의 의도가 불명확하면 반드시 되물어본다.
- 불완전한 데이터(참조 문서 없음 등)를 감추지 말고, 가정으로 채워도 될지 먼저 물어본다.
- 중간 결과(목차, 섹션 요약, 이미지 프롬프트)는 가능한 한 공유하여 사용자 개입을 허용한다.
