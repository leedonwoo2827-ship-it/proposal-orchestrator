# 제안서 오케스트레이션 워크플로

## Purpose
사용자의 자연어 요청을 해석하여 제안서 작성, 이미지 생성, 진행 상황 보고를 단계적으로 수행한다.

## When to Use
- 사용자가 제안서 작성을 요청할 때
- 제안서와 이미지를 동시에 생성해야 할 때
- 프로젝트 진행 상황을 확인하거나 검토할 때

## 핵심 원칙: 타임아웃 방지

> ⚠️ **Cowork 응답 시간 제한을 항상 고려하라.**
>
> 1. **한 번의 응답에서 MCP 툴을 2개 이상 호출하지 마라.**
> 2. **각 단계를 짧게 처리하고 사용자 확인을 받아라.**
> 3. **10만자 이상의 대량 텍스트를 한 번에 생성하지 마라.** 섹션별로 나눠서 작성하라.
> 4. **파일 분석도 한 번에 1~2개만.** 여러 파일을 동시에 분석하지 마라.
> 5. **첫 응답은 무조건 빠르게.** 폴더 확인 + 계획 제시만 하고 멈춰라.

## 입력 파싱 규칙

사용자 메시지에서 다음 요소를 추출한다:

### 작업 유형
- **제안서 전체 생성**: "제안서 만들어줘", "양식으로 제안서 작성해줘"
- **뼈대/구조만**: "뼈대 만들어줘", "목차 잡아줘", "구조 생성해줘"
- **일부 섹션 보완**: "시장 분석 부분 보완해줘", "결론 다시 써줘"
- **이미지 생성**: "표지 이미지 만들어줘", "슬라이드 이미지 3장"
- **워치/점검**: "진행 상황 보여줘", "빠진 섹션 확인해줘"
- **HWPX 출력**: "HWPX로 저장해줘", "한글 파일로 내보내줘"

### 프로젝트 식별
- project_dir 경로 (예: `C:\Users\ubion\Documents\proposals\260311-naa`)
- "이전에 만들었던 ~ 제안서" 같은 설명

### 텍스트 옵션
- 목표 글자 수: 5만 / 10만 / 15만 / 20만자
- 톤/스타일: 금융, 공공, 교육, IT, 건설 등

### 이미지 옵션
- 생성 개수, 스타일, 용도 (표지/슬라이드/요약/결론)

## 필수/선택 입력

| 항목 | 필수 여부 | 비고 |
|------|----------|------|
| project_dir | **필수** | 프로젝트 폴더 경로 |
| templates/ | **필수** | 양식 파일 (RFP, 양식 등) |
| rawdata/ | 선택 | 없으면 건너뜀, 양식 기반으로만 작성 |
| references/ | 선택 | 없으면 건너뜀 |

## 오케스트레이션 규칙

### 제안서 전체 생성 워크플로

**반드시 단계별 대화 방식으로 진행한다. 한 번에 전체를 실행하지 않는다.**

```
[1단계] 폴더 확인 + 계획 제시 → 사용자 확인 대기
  ↓ 사용자: "진행해"
[2단계] 양식 분석 → 목차 도출 → 사용자 확인 대기
  ↓ 사용자: "좋아" / 수정 요청
[3단계] 본문 작성 (섹션 1~2개씩) → 중간 보고 → 사용자 확인 대기
  ↓ 반복
[4단계] 이미지 프롬프트 제시 → 사용자 확인 → 생성
  ↓
[5단계] HWPX 출력
```

각 단계의 상세:

1. **폴더 확인**: filesystem으로 project_dir 하위 폴더/파일 확인. 즉시 보고.
2. **양식 분석**: templates/ 파일을 읽고 목차/섹션 구조 도출. rawdata 없으면 "양식 기반으로 구조를 잡겠습니다"라고 안내.
3. **본문 작성**: 섹션 1~2개씩 작성. 출처 마커 적용 (`{{red:}}`, `{{green:}}`). 볼드 미사용.
4. **이미지 생성**: text2img-mcp 호출. project_dir 전달하여 images/ 폴더에 저장.
5. **HWPX 출력**: hwpx-writer 호출. project_dir + image_paths 전달하여 output/ 폴더에 저장.

### 뼈대/구조만 요청한 경우

1단계 + 2단계만 수행하고 멈춘다. 본문 작성은 사용자가 별도로 요청할 때 진행한다.

### 이미지 중심 워크플로
1. 텍스트 요약 또는 기존 내용 활용
2. 이미지 프롬프트 설계 → 사용자 확인
3. text2img-mcp로 생성

### 워치/점검 워크플로
1. project_dir의 output/, images/ 폴더 확인
2. 완료/미완료 상태 보고
3. 보완 작업 제안

## 대화 스타일
- 각 단계를 짧게 보고한다
- 결과 반환 시 다음 선택지를 함께 제안한다
- **응답은 항상 간결하게.** 불필요한 설명을 붙이지 않는다.

## 주의사항
- 사용자의 의도가 불명확하면 반드시 되물어본다
- rawdata나 references가 없어도 에러로 처리하지 않는다. 있는 자료만으로 진행한다.
- 중간 결과(목차, 섹션 요약, 이미지 프롬프트)는 가능한 한 공유하여 사용자 개입을 허용한다
