# 제안서 스타일 가이드

## Purpose
제안서 작성 시 일관된 품질과 톤을 유지하기 위한 스타일 규칙을 정의한다.

## When to Use
- 제안서 본문을 작성할 때
- 섹션별 톤/스타일을 결정할 때
- 글자 수 배분을 계획할 때

## 글자 수 배분 기준

### 5만자 제안서
| 섹션 | 비율 | 대략 글자 수 |
|------|------|-------------|
| 개요/서론 | 10% | 5,000자 |
| 현황 분석 | 15% | 7,500자 |
| 제안 내용 (핵심) | 40% | 20,000자 |
| 추진 체계/일정 | 15% | 7,500자 |
| 기대 효과 | 10% | 5,000자 |
| 부록/참고 | 10% | 5,000자 |

### 10만자 제안서
위 비율을 동일하게 적용하되, 각 섹션의 세부 항목을 2~3배 확장한다.

### 15만~20만자 제안서
핵심 제안 내용의 비중을 45%까지 확대하고, 사례 분석/벤치마크 섹션을 추가한다.

## 분야별 톤 가이드

### 금융/투자
- 데이터 중심, 수치 근거 강조
- 리스크 관리 관점 포함
- 보수적이고 신뢰감 있는 어조

### 공공/행정
- 법적 근거, 정책 연계성 강조
- 공익적 관점, 사회적 가치 언급
- 격식체 사용, 객관적 서술

### IT/기술
- 기술 스택, 아키텍처 상세 기술
- 확장성, 보안, 성능 지표 포함
- 전문 용어 적절히 활용

### 교육
- 학습 효과, 교육 방법론 강조
- 사례 중심 설명
- 이해하기 쉬운 표현 사용

### 건설/엔지니어링
- 시방서, 공법, 안전 관리 강조
- 도면/설계 기준 참조
- 단가 및 수량 산출 근거 포함

## 출처별 색상 표시 규칙

본문 작성 시 텍스트의 출처에 따라 색상을 구분하여 표시한다.
이를 통해 검토자가 어떤 내용이 원본 데이터에서 왔고, 어떤 내용이 참조 문서에서 왔는지 즉시 파악할 수 있다.

### 색상 규칙
| 출처 | 마커 형식 | 색상 | 용도 |
|------|----------|------|------|
| rawdata (로데이터/원본 데이터) | `{{red:구절}}` | 적색 | RFP 원문, 고객 제공 데이터, 원본 요구사항 등에서 직접 가져온 텍스트 |
| 참조문서 (기존 제안서/레퍼런스) | `{{green:구절}}` | 녹색 | 기존 제안서, 참조 문서, 사례집 등에서 가져온 텍스트 |
| AI 생성 텍스트 | 마커 없음 | 기본(검정) | AI가 새로 작성한 텍스트 |

### 태깅 방식
본문 작성 시 출처별 텍스트를 다음 인라인 마커로 감싼다:
- rawdata 출처: `{{red:텍스트}}`
- 참조문서 출처: `{{green:텍스트}}`
- AI 생성: 마커 없음 (기본)

hwpx-writer가 이 마커를 인식하여 해당 색상의 글자로 자동 변환한다.

**작성 예시:**
```
본 훈련과정은 {{red:AI 기초(10차시), AI 심화(10차시)}} 등 총 5개 과정을 통합하였으며,
{{green:SRT 기반 자막 데이터로 전 차시가 체계적으로 구조화}}되어 있습니다.
```

### 적용 원칙
1. 원문을 그대로 인용하거나 최소 수정만 한 경우에만 색상 태깅한다.
2. 원문을 기반으로 크게 재작성한 경우는 기본색(AI 생성)으로 처리한다.
3. 한 문단 안에서 출처가 혼합된 경우, 구절 단위로 태깅한다.
4. 마커는 구(phrase) 또는 절(clause) 단위로 감싼다 — 문장 전체를 감싸지 않는다.

## 볼드(강조) 처리 규칙

> ⚠️ **현재 hwpx-writer는 볼드/이탤릭 렌더링을 지원하지 않는다.**
> 마크다운의 `**텍스트**`, `*텍스트*` 는 HWPX 변환 시 별표가 제거되고 일반 텍스트로 출력된다.

### 강조가 필요한 경우 대체 방법
- **색상 마커 활용**: `{{red:핵심 수치}}` 또는 `{{green:핵심 근거}}`
- **헤딩 레벨(lv) 활용**: `#### ● 항목명` 형태로 소항목으로 분리하여 시각적 구분
- **기호 활용**: `― 세부 설명` 형태로 들여쓰기 구분

## 공통 작성 원칙
1. **두괄식 서술**: 핵심 결론을 먼저 제시하고 근거를 뒤에 배치
2. **구체적 수치 활용**: "많은" 대신 "전년 대비 23% 증가"
3. **표/도표 활용**: 비교 항목이 3개 이상이면 표로 정리
4. **단락 길이**: 한 단락당 3~5문장, 200~400자 내외
5. **능동태 사용**: "~로 판단된다" 대신 "~로 판단한다"
