# /proposal-orchestrator:create-proposal

제안서 전체를 생성하는 커맨드입니다. 양식 분석부터 이미지 생성, HWPX 출력까지 전 과정을 오케스트레이션합니다.

## 실행 절차

사용자가 이 커맨드를 호출하면 다음 순서로 진행하세요:

### 1단계: 요청 확인
사용자에게 다음 항목을 확인하세요:
- **양식 파일**: 제안서 양식이 제공되었는지 확인
- **참조 제안서**: 참조할 기존 제안서나 로데이터가 있는지 확인
- **목표 글자 수**: 5만 / 10만 / 15만 / 20만자 중 선택 (기본값: 10만자)
- **이미지 개수**: 필요한 이미지 수와 용도 (기본값: 표지 1장 + 슬라이드 3장)
- **분야/톤**: 금융, 공공, IT, 교육, 건설 등
- **HWPX 출력 여부**: 최종 파일로 내보낼지 여부

확인되지 않은 항목은 기본값을 제안하고 사용자의 동의를 받으세요.

### 2단계: 프로젝트 생성
- project_id를 생성합니다 (형식: PROP-YYYYMMDD-NNN)
- 사용자에게 project_id를 알려줍니다

### 3단계: 양식 분석
- 제공된 양식 파일을 분석합니다
- 섹션 구조와 목차를 도출합니다
- 도출된 목차를 사용자에게 보여주고 확인 받습니다

### 4단계: 참조 데이터 분석
- 참조 제안서 또는 로데이터를 분석합니다
- 섹션별 핵심 포인트를 추출합니다
- 추출 결과를 간단히 요약하여 보여줍니다

### 5단계: 본문 작성
- 목표 글자 수에 맞춰 섹션별 본문을 작성합니다
- proposal-writing 스킬을 활용합니다
- company-data와 rfp-analysis 스킬로 데이터를 보강합니다
- 대규모 제안서의 경우 서론→본론→결론 순으로 나누어 작성합니다
- 각 섹션 완료 시 진행 상황을 보고합니다
- **출처별 마커**: rawdata에서 가져온 텍스트는 `{{red:텍스트}}`, 참조문서에서 가져온 텍스트는 `{{green:텍스트}}`로 마킹합니다
- **볼드 미사용**: hwpx-writer가 볼드 렌더링을 지원하지 않습니다. 강조는 색상 마커(`{{red:}}`, `{{green:}}`) 또는 헤딩 레벨(lv)로 대체합니다

### 6단계: 이미지 생성
- 각 이미지의 용도와 프롬프트를 설계합니다
- text2img-mcp를 호출하여 이미지를 생성합니다
- 생성된 이미지에 대한 설명/캡션을 제안합니다

### 7단계: 결과 묶기 및 출력
- HWPX 출력이 요청된 경우, hwpx-writer를 호출합니다
- 본문 텍스트 + 이미지 정보 + 스타일 옵션을 전달합니다
- `{{red:}}` / `{{green:}}` 마커는 hwpx-writer가 자동으로 색상으로 변환합니다. 별도 변환 작업 불필요.
  - proposal-style-guide의 색상 표시 규칙을 준수합니다
- 최종 파일 경로를 사용자에게 안내합니다

### 8단계: 결과 요약
- 전체 작업 결과를 요약합니다
- 다음 수정/보완 단계를 제안합니다

## 사용 예시
```
/proposal-orchestrator:create-proposal
이 양식이랑 참조제안서로 10만자 제안서 만들어줘.
표지 이미지 1장, 핵심 슬라이드용 이미지 4장 같이 만들어줘.
```
