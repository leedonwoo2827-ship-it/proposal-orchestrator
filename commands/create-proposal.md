# /proposal-orchestrator:create-proposal

제안서 전체를 생성하는 커맨드입니다.

## 핵심 원칙: 단계별 대화

> ⚠️ **절대로 전체 과정을 한 번에 실행하지 마라.**
> Cowork은 응답 시간 제한이 있다. 각 단계를 짧게 처리하고 사용자 확인을 받은 뒤 다음 단계로 넘어가라.
> 한 번의 응답에서 MCP 툴을 2개 이상 호출하지 마라.

## 실행 절차

### 1단계: 폴더 확인 및 계획 수립 (첫 응답에서 반드시 완료)

사용자가 이 커맨드를 호출하면 **즉시 다음만 수행**하고 응답하라:

1. project_dir 경로 확인
2. filesystem으로 폴더 구조 확인 (templates/, rawdata/, references/ 존재 여부와 파일 목록)
3. 다음 형식으로 **짧게** 보고:

```
[폴더 확인 완료]
- 프로젝트: {폴더명}
- templates: {파일 수}개 ({파일명 나열})
- rawdata: {파일 수}개 또는 "없음"
- references: {파일 수}개 또는 "없음"

[작업 계획]
1. 양식 분석 → 목차/뼈대 생성
2. 본문 작성 (목표: {N}만자)
3. 이미지 생성 ({N}장)
4. HWPX 출력

진행할까요?
```

**이 응답 이후 반드시 멈추고 사용자 확인을 기다려라.**

### 2단계: 양식 분석 → 목차 도출

사용자가 "진행해" 또는 확인하면:
- templates 폴더의 양식 파일을 분석한다
- 섹션 구조와 목차를 도출한다
- 도출된 목차를 보여주고 확인을 받는다

**rawdata가 없는 경우**: 건너뛰고 templates만으로 뼈대를 생성한다. "rawdata가 없어서 양식 기반으로만 구조를 잡겠습니다"라고 안내한다.

**references가 없는 경우**: 정상이다. references는 선택사항이다.

### 3단계: 본문 작성 + 이미지 위치 지정

사용자가 목차를 승인하면:
- 목표 글자 수에 맞춰 섹션별 본문을 작성한다
- **한 번에 1~2개 섹션만 작성**하고 중간 보고한다
- 출처별 마커: rawdata → `{{red:텍스트}}`, 참조문서 → `{{green:텍스트}}`
- 마크다운 볼드(`**텍스트**`)는 사용하지 않는다
- **이미지 삽입 위치를 본문 중간에 마크다운 태그로 지정한다:**
  ```
  ![이미지 설명 - 프롬프트 힌트](images/placeholder-01.png)
  ```
  - 각 섹션을 작성하면서 이미지가 필요한 위치에 `![](...)` 태그를 배치
  - 파일명은 `placeholder-01.png`, `placeholder-02.png` 등 임시 이름 사용
  - 설명란에 이미지 컨셉/프롬프트 힌트를 기록

### 4단계: 이미지 생성 및 경로 교체

본문의 이미지 태그를 기반으로:
1. 각 `![설명](images/placeholder-XX.png)` 태그의 설명을 참고하여 이미지 프롬프트 설계
2. 사용자 확인 후 text2img-mcp로 생성 (`project_dir` 전달 → images/ 폴더 저장)
3. 생성된 실제 파일명으로 마크다운의 placeholder 경로를 교체:
   ```
   ![사업 구조도](images/1773278147172.png)
   ```

> ⚠️ **image_paths 파라미터를 사용하지 마라.** 이미지는 반드시 마크다운 본문의 `![](...)` 태그로만 삽입한다. image_paths로 넣으면 문서 끝에 몰리게 된다.

### 5단계: HWPX 출력

- hwpx-writer의 `convert_md_to_hwpx` 호출
- `project_dir` 파라미터로 output/ 폴더에 자동 저장
- 마크다운에 포함된 `![](...)` 태그가 자동으로 해당 위치에 이미지 삽입됨
- **image_paths 파라미터는 사용하지 않는다**

## 필수 입력과 선택 입력

| 항목 | 필수 여부 | 기본값 |
|------|----------|--------|
| project_dir (폴더 경로) | **필수** | - |
| templates/ 폴더 | **필수** | - |
| rawdata/ 폴더 | 선택 | 없으면 건너뜀 |
| references/ 폴더 | 선택 | 없으면 건너뜀 |
| 목표 글자 수 | 선택 | 10만자 |
| 이미지 개수 | 선택 | 표지 1 + 슬라이드 3 |

## 사용 예시
```
/proposal-orchestrator:create-proposal
C:\Users\ubion\Documents\proposals\260311-naa 폴더로 작업해줘.
templates 폴더의 양식으로 제안서 뼈대 만들어줘.
```
