# Proposal Orchestrator 사용법

Claude Desktop 채팅창에서 제안서 작성을 자동화하는 실전 사용 가이드입니다.
설치가 완료된 상태를 전제합니다. 설치는 `proposal-orchestrator-설치법.md`를 참고하세요.

---

## 1. 작업 폴더에 파일 준비하기

새 프로젝트를 시작할 때마다 **날짜-폴더**를 만들고, 그 안에 3개 입력 폴더를 채웁니다.

### 폴더 만들기

```powershell
# 예: 오늘 스마트시티 프로젝트 시작
mkdir C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티\rawdata
mkdir C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티\references
mkdir C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티\templates
```

만들면 이런 구조가 됩니다:

```
C:\Users\leedonwoo\Documents\proposals\
└── 20260303-스마트시티\
    ├── rawdata\        ← 1. 원본 로데이터
    ├── references\     ← 2. 참조 제안서
    ├── templates\      ← 3. 양식 파일
    └── output\         ← (결과물 - Claude가 자동 생성)
```

### rawdata\ — 원본 로데이터

제안서의 근거가 되는 수치/팩트 자료를 넣습니다.

| 파일 종류 | 예시 |
|-----------|------|
| 매출/실적 데이터 | `회사실적_2025.xlsx`, `분기매출.csv` |
| 시장/산업 통계 | `IT시장전망_2026.pdf`, `산업통계.xlsx` |
| 기술 자료 | `시스템아키텍처.pptx`, `API명세서.pdf` |
| 내부 보고서 | `사업현황보고.docx`, `경쟁분석.pdf` |
| 고객 요구사항 | `고객미팅_메모.docx`, `VOC정리.xlsx` |

> 파일이 많아도 괜찮습니다. Claude가 필요한 파일만 골라서 분석합니다.
> 단, 파일명을 알아보기 쉽게 지으면 정확도가 높아집니다.
>
> 하위 폴더로 분류해도 됩니다:
> ```
> rawdata\
> ├── 재무\
> ├── 기술\
> └── 시장\
> ```

### references\ — 참조 제안서

이전에 작성했거나 벤치마크할 제안서를 넣습니다.

| 파일 종류 | 예시 |
|-----------|------|
| 수주한 제안서 | `A사_클라우드전환_제안서.pdf` |
| 경쟁사 제안서 | `B사_유사사업_제안서.pdf` |
| 우수 사례 | `최우수_스마트시티_제안서.hwpx` |
| 유사 분야 샘플 | `금융_디지털전환_사례.pdf` |

> Claude는 참조 제안서에서 구조, 논리 전개, 표현 방식을 학습합니다.
> 품질 좋은 참조 제안서를 넣을수록 결과물이 좋아집니다.

### templates\ — 양식 파일

발주처나 고객이 제공한 공식 양식을 넣습니다.

| 파일 종류 | 예시 |
|-----------|------|
| 제안서 양식 | `공공_표준양식_2026.hwpx` |
| RFP 본문 | `RFP_스마트시티.pdf` |
| 평가 기준표 | `기술평가_배점표.xlsx` |
| 제출 서류 목록 | `제출서류_안내.pdf` |

> 양식은 섹션 구조(목차)를 도출하는 핵심 입력입니다.
> 양식이 없으면 Claude가 범용 구조를 제안합니다.

## 2. 명령어 목록

| 명령어 | 기능 |
|--------|------|
| `/proposal-orchestrator:create-proposal` | 제안서 전체 생성 |
| `/proposal-orchestrator:generate-images` | 이미지 생성 |
| `/proposal-orchestrator:watch-status` | 진행 상황 점검 |
| `/proposal-orchestrator:export-hwpx` | HWPX 파일 출력 |

> 명령어 없이 자연어로 말해도 Claude가 알아서 실행합니다.

## 3. 제안서 전체 생성

### 기본 사용법

채팅창에 명령어와 함께 프로젝트 폴더 경로를 지정합니다:

```
/proposal-orchestrator:create-proposal
C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티 폴더로 작업해줘.
templates 폴더의 양식으로, references 폴더의 참조제안서를 참고해서
10만자 제안서 만들어줘.
rawdata 폴더의 데이터도 활용해줘.
표지 이미지 1장, 슬라이드 이미지 4장도 같이 만들어줘.
분야는 IT/디지털 전환, 톤은 전문적으로.
```

### 파일을 직접 첨부하는 경우

파일을 채팅창에 드래그 앤 드롭하고 말합니다:

```
이 양식이랑 참조제안서로 10만자 제안서 만들어줘.
표지 이미지 1장, 핵심 슬라이드용 이미지 4장도 같이 만들어줘.
```

### Claude가 진행하는 과정

```
1단계: 요청 확인
  → "10만자, IT 분야, 이미지 5장(표지 1 + 슬라이드 4)으로 진행할까요?"
  → (사용자 확인 대기)

2단계: 프로젝트 생성
  → "Project ID: PROP-20260303-001을 생성했습니다."

3단계: 양식 분석
  → "양식에서 7개 섹션을 도출했습니다. 목차를 확인해 주세요."
  → [목차 표시]
  → (사용자 확인 대기)

4단계: rawdata + 참조제안서 분석
  → "20260303-스마트시티\rawdata 에서 5개 파일을 분석했습니다."
  → "20260303-스마트시티\references 에서 참조제안서 3개의 핵심 포인트를 추출했습니다."

5단계: 본문 작성
  → "1/7 개요 섹션 작성 완료 (10,200자)"
  → "2/7 현황 분석 작성 완료 (15,100자)"
  → ... (섹션별 진행 보고)

6단계: 이미지 생성
  → "표지 이미지 프롬프트를 확인해 주세요: [프롬프트]"
  → "이미지 1/5 생성 완료"
  → ... (이미지별 진행 보고)

7단계: HWPX 출력
  → "제안서.hwpx 파일이 C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티\output\ 에 생성되었습니다."

8단계: 결과 요약
  → "전체 102,400자, 이미지 5장 포함. 수정이 필요한 부분이 있으신가요?"
```

### 중간에 개입하기

언제든 Claude에게 말해서 흐름을 바꿀 수 있습니다:

```
여기서 멈춰.
3번 섹션 목차를 좀 바꾸고 싶어.
```

```
rawdata 폴더에 파일 하나 더 추가했어.
C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티\rawdata\추가자료.xlsx 도 참고해줘.
```

## 4. 이미지만 생성

### 새로 생성

```
/proposal-orchestrator:generate-images
이 제안서 요약을 기반으로 썸네일 이미지 3개 만들어 줘.
분위기는 금융/투자 느낌으로.
```

### 기존 프로젝트에 추가

```
아까 만든 제안서에 인포그래픽 이미지 2장 추가해줘.
시장 분석이랑 수익 모델 섹션에 넣을 거야.
```

### 재생성

```
표지 이미지가 마음에 안 들어.
좀 더 모던하고 깔끔한 느낌으로 다시 만들어줘.
```

### 이미지 프롬프트 조정

Claude는 이미지 생성 전에 프롬프트를 보여줍니다:

```
[Claude] 표지 이미지 프롬프트:
"A professional cover image for a digital transformation proposal,
modern minimalist design, blue and white corporate color scheme..."

이 프롬프트로 진행할까요?
```

여기서 조정할 수 있습니다:

```
색상을 남색+금색으로 바꿔줘.
텍스트 넣을 공간을 좀 더 넓혀줘.
```

## 5. 진행 상황 점검 (워치)

```
/proposal-orchestrator:watch-status
지금까지 만든 제안서 구조와 이미지 목록을 정리해서 보여줘.
```

### Claude 응답 예시

```
[프로젝트 종합 현황]
- 프로젝트: 20260303-스마트시티
- 작업 폴더: C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티\
- 목표: 10만자 제안서 + 이미지 5장

[텍스트 작성 현황]
- 완료: 1. 개요, 2. 현황 분석, 3. 제안 내용
- 작성 중: 4. 기술 아키텍처 (약 60%)
- 미작성: 5. 추진 일정, 6. 기대 효과, 7. 부록
- 현재: 약 62,000자 / 목표 100,000자

[이미지 현황]
- 표지: 1장 완료
- 슬라이드: 2/4장 완료 (현황 분석, 제안 내용)
- 미생성: 기술 아키텍처, 기대 효과

[제안]
- 기술 아키텍처 섹션 완성을 먼저 진행할까요?
- 미생성 이미지 2장을 함께 만들까요?
```

### 점검 후 이어서 작업

```
미작성 섹션 전부 이어서 작성해줘.
```

```
빠진 이미지 2장도 만들어줘.
```

## 8. 자연어 요청 매핑

명령어를 외우지 않아도 됩니다. 자연어로 말하면 Claude가 알아서 처리합니다.

| 이렇게 말하면 | Claude가 하는 일 |
|---------------|-----------------|
| "제안서 만들어줘" | 전체 생성 워크플로 시작 |
| "이미지 3장 만들어줘" | 이미지 생성 |
| "어디까지 했어?" | 진행 상황 점검 |
| "HWPX로 저장해줘" | HWPX 출력 |
| "시장 분석 부분 보완해줘" | 해당 섹션 부분 수정 |
| "표지 이미지 다시 만들어줘" | 이미지 재생성 |
| "빠진 섹션 확인해줘" | 워치 + 보완 제안 |
| "rawdata에 파일 추가했어" | 추가 데이터 반영 |

## 9. 결과물의 색상과 서식

### 출처별 글자 색상

생성된 제안서에서는 텍스트의 출처에 따라 글자 색상이 다르게 표시됩니다:

| 글자 색상 | 의미 |
|-----------|------|
| **적색 (빨간색)** | rawdata에서 가져온 텍스트 (매출 데이터, 통계, 고객 요구사항 등 원본 수치/팩트) |
| **녹색 (초록색)** | 참조 제안서에서 가져온 텍스트 (기존 제안서의 표현, 구조, 사례 등) |
| **검정색 (기본)** | AI가 새로 작성한 텍스트 |

> 이 색상 구분을 통해 검토 시 어느 내용이 원본 데이터 기반인지,
> 참조 문서에서 온 것인지, AI가 생성한 것인지 한눈에 파악할 수 있습니다.

### 볼드(굵은 글씨) 표시

섹션 제목, 핵심 키워드 등 강조가 필요한 부분은 **굵은 글씨**로 표시됩니다.
HWPX 파일에서도 동일하게 Bold 서식이 적용됩니다.

## 10. 팁

### rawdata가 많을 때
- 파일명을 명확하게 지으세요 (예: `2025_매출실적_분기별.xlsx`)
- Claude에게 특정 파일만 지정할 수도 있습니다:
  ```
  rawdata 폴더의 시장분석_보고서.pdf 랑 산업통계.xlsx 만 참고해줘.
  나머지는 무시해도 돼.
  ```

### 대용량 제안서 (10만자 이상)
- 섹션별로 나누어 순차 작성됩니다
- 각 섹션 완료 시 진행 보고를 받습니다
- 중간에 "여기서 멈춰" 또는 "이 부분 수정해줘"로 개입 가능

### 여러 프로젝트 관리
- 프로젝트마다 날짜-폴더를 만들어 관리합니다:
  ```
  C:\Users\leedonwoo\Documents\proposals\
  ├── 20260303-스마트시티\
  ├── 20260305-디지털전환\
  └── 20260310-클라우드이전\
  ```
- 새 대화에서 이어 작업하려면:
  ```
  C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티 프로젝트 이어서 진행해줘.
  ```
- 각 프로젝트의 결과물은 해당 폴더의 `output\`에 저장됩니다

### 참조 제안서 활용 극대화
- 수주에 성공한 제안서를 넣으면 효과적입니다
- 같은 분야(금융, 공공, IT 등)의 참조를 넣으면 톤이 맞습니다
- 여러 개를 넣으면 Claude가 공통 패턴을 추출합니다

### 다른 프로젝트의 자료 참조하기
- 이전 프로젝트의 rawdata나 references를 활용할 수도 있습니다:
  ```
  이번 제안서에 C:\Users\leedonwoo\Documents\proposals\20260303-스마트시티\references 폴더의
  참조제안서도 같이 참고해줘.
  ```
